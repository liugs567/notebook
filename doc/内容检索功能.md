## 全站内容检索功能描述

### 1. 功能概述

在博客系统中新增**全站正文内容检索**能力。用户通过全局入口打开搜索面板，输入关键词后，系统在所有文章的 Markdown 正文中进行匹配，并以「标题 + 上下文片段」的形式展示结果；点击结果可跳转到文章阅读页，并自动滚动、定位到命中位置，同时高亮显示搜索词。

该功能与现有列表页的**标题搜索**互补：列表搜索仅匹配 `title`，本功能面向**正文内容**的全文检索。

---

### 2. 使用场景

- 记得文章里写过某个概念、命令或句子，但不记得是哪篇、在哪个位置
- 需要在多篇笔记中快速查找同一技术点、代码片段或引用
- 从任意页面（列表、阅读、编辑等）发起检索，不打断当前浏览上下文

---

### 3. 交互流程

```mermaid
flowchart LR
  A[点击全局搜索按钮] --> B[全屏模糊遮罩 + 搜索栏]
  B --> C[输入关键词]
  C --> D[展示匹配结果列表]
  D --> E[点击某条结果]
  E --> F[跳转文章阅读页]
  F --> G[滚动定位到命中区域并高亮]
```

1. 用户在页面固定位置点击「内容检索」按钮（建议图标 + 快捷键，如 `⌘/Ctrl + K`）
2. 当前页面被**全屏半透明遮罩 + 背景模糊**覆盖，中央或顶部出现搜索输入框
3. 用户输入关键词，系统实时或防抖后返回匹配结果
4. 结果列表展示：文章标题、命中片段（前后各若干字符的上下文）、可选命中次数
5. 用户点击某条结果 → 关闭搜索面板 → 跳转至 `/blog/view/:id`，页面加载完成后**自动滚动到对应位置**，并对关键词做**临时高亮**
6. 按 `Esc` 或点击遮罩空白处关闭搜索面板，恢复原页面

### 4. 界面与交互细节

#### 4.1 入口

| 项目 | 说明 |
|------|------|
| 位置 | 全局可用（如顶部导航栏、列表页工具栏），与现有「标题搜索」区分 |
| 样式 | 搜索图标按钮，hover 提示「内容检索」 |
| 快捷键 | 建议 `Ctrl/Cmd + K` 打开，`Esc` 关闭 |

#### 4.2 全屏搜索面板

| 项目 | 说明 |
|------|------|
| 遮罩 | 覆盖视口，`backdrop-filter: blur()` 模糊底层内容 |
| 搜索栏 | 大号输入框，placeholder 如「搜索全部文章内容…」 |
| 空态 | 未输入时展示提示：「输入关键词搜索正文内容」 |
| 加载态 | 检索中显示 loading |
| 无结果 | 「未找到包含「xxx」的内容」 |

#### 4.3 结果列表

每条结果包含：

| 字段 | 说明 |
|------|------|
| 文章标题 | 主标题，可点击 |
| 上下文片段 | 命中词前后各约 40–80 字符（可配置），超出以 `…` 省略 |
| 关键词高亮 | 片段内匹配文字用醒目样式（如黄色背景 / 主题色） |
| 辅助信息 | 可选：更新时间、标签、命中序号（同一文章多处命中时拆成多条） |

**同一文章多处命中**：每条命中单独一行，标题相同、片段不同；或标题合并显示「共 N 处匹配」，展开后列出各片段（二选一，实现时择一即可）。

#### 4.4 跳转与定位

| 项目 | 说明 |
|------|------|
| 路由 | 跳转至现有阅读页 `BlogView`（`/blog/view/:id`） |
| 定位参数 | URL 携带定位信息，如 `?q=关键词&offset=1234` 或 `?highlight=xxx&anchor=block-id` |
| 滚动行为 | 页面渲染完成后 `scrollIntoView`，带适当 `scroll-margin-top` 避免被顶栏遮挡 |
| 高亮 | 阅读页正文中对匹配词做临时高亮（如 `<mark>`），若干秒后淡出或直到用户再次滚动/点击 |
| 失败兜底 | 无法精确定位时，仍打开文章并在页内做一次全文高亮 |

### 5. 检索范围与规则

#### 5.1 检索范围

- **正文**：`content.md` 的 Markdown 源文本（检索前 strip 掉纯语法符号的影响，或同时检索渲染后纯文本，以实现为准）
- **文章范围**：与列表一致，包含 `saved/` 与 `temp_drafts/` 中的全部文章；是否排除未发布草稿可配置（默认可搜全部，与列表「标题搜索」策略一致）
- **不检索**：图片 alt 以外的二进制资源、仅存在于 assets 的文件名（除非后续扩展）

#### 5.2 匹配规则

| 规则 | 说明 |
|------|------|
| 默认 | 子串匹配，不区分大小写（中文按字符，英文 `toLowerCase`） |
| 可选增强 | 多关键词空格分词（AND）；引号包裹精确短语 |
| 最小长度 | 建议 ≥ 2 字符，避免单字误匹配过多 |
| 结果上限 | 单次最多返回 50–100 条，超出提示「结果过多，请缩小关键词」 |

#### 5.3 片段提取

- 以首次命中位置为中心，向前后扩展至词边界或固定字符数
- 去除 Markdown 语法噪音（如 `#`、`` ` ``、链接 URL）再展示，提升可读性
- 保留足够上下文，使用户能判断是否为要找的内容

### 6. 技术实现要点（供开发参考）

#### 6.1 后端

- 新增 API：`GET /api/blog/search?q=关键词&limit=20`
- 响应结构示例：

```json
{
  "items": [
    {
      "id": "uuid",
      "title": "文章标题",
      "snippet": "…前后文**命中词**前后文…",
      "matchIndex": 0,
      "offset": 1234,
      "updateTime": 1710000000000
    }
  ],
  "total": 12,
  "keyword": "关键词"
}
```

- 实现方式（择一或组合）：
  - **MVP**：遍历索引 + 按需读取 `content.md` 内存匹配（文章量小时足够）
  - **进阶**：索引重建时写入 `search-index.json`（id、title、plainText、offsets），或内存缓存正文摘要
- 与现有 `listArticles` 的 `keyword`（仅 title）分离，避免行为混淆

#### 6.2 前端

- 新增全局组件：`ContentSearchOverlay.vue`（遮罩 + 输入 + 结果列表）
- 在 `App.vue` 或布局层挂载，通过 composable / provide 控制显隐
- 调用 `searchContent(q)` API，防抖 300ms
- 跳转：`router.push({ path: '/blog/view/:id', query: { q, offset } })`
- `BlogView.vue` 扩展：`onMounted` 读取 query，渲染完成后定位并高亮（可在 MdPreview 渲染 DOM 中查找文本节点或使用 md-editor 提供的锚点）

#### 6.3 定位策略建议

| 方案 | 优点 | 缺点 |
|------|------|------|
| 字符 offset | 实现简单，与 Markdown 源一致 | 渲染后 DOM 与源文本位置可能偏差 |
| 块级 anchor | 定位稳定 | 需在渲染时给标题/段落加 id |
| 纯文本搜索 DOM | 跳转后准确 | 大文章可能略慢 |

**推荐**：MVP 用 `offset` + 渲染后在预览区 DOM 内查找包含关键词的第一个可见节点；后续可改为段落 hash / heading id。

### 7. 非功能需求

| 项目 | 要求 |
|------|------|
| 性能 | 100 篇以内文章，检索响应 < 500ms（本地 Node 读文件） |
| 体验 | 打开/关闭面板有 200ms 过渡动画；支持键盘上下选择结果、Enter 跳转 |
| 无障碍 | 输入框自动 focus；Esc 关闭；结果项可 Tab 聚焦 |
| 样式 | 与现有玻璃拟态 / 暗色主题保持一致 |

---

### 8. 验收标准

- [ ] 点击按钮或快捷键，出现全屏模糊遮罩与搜索栏
- [ ] 输入关键词后，能检索到**正文**中的匹配（不仅标题）
- [ ] 结果列表展示文章标题 + 含关键词的上下文片段，关键词高亮
- [ ] 点击结果跳转到对应文章阅读页
- [ ] 阅读页自动滚动到命中区域附近，关键词可见高亮
- [ ] `Esc` / 点击遮罩可关闭搜索面板，不影响原页面状态
- [ ] 无匹配、加载中、结果过多等状态有明确提示

### 9. 与现有功能的关系

| 现有能力 | 本功能 |
|----------|--------|
| 列表页标题搜索（`keyword` → `title`） | 保留不变 |
| 阅读页大纲目录（MdCatalog） | 互补：大纲按结构导航，本功能按全文关键词导航 |
| 文章详情 API（`getArticleDetail`） | 跳转定位时复用，搜索 API 单独提供 |
