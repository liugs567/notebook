# 个人博客功能需求文档（Node.js 本地文件版 · 完整版）
## 文档说明
本文档为**完整可直接开发**的个人博客需求文档，基于 Vue + md-editor-v3 + Node.js 实现，**无数据库、纯本地文件存储**，包含博客写作、列表、保存、自动草稿、图片管理等全功能。所有界面、交互、逻辑、接口、目录结构、服务端规则统一收口，无需额外补充。

适用场景：前端 Vue3 + 后端 Node.js（Express/Koa）+ 本地文件存储

---

# 一、项目概述
## 1.1 基本信息
- 模块名称：博客文章写作与管理模块
- 核心编辑器：md-editor-v3（Markdown 编辑、预览、代码高亮、图片上传）
- 存储方案：Node.js 本地文件系统（无数据库），在**可配置根目录**下分两个总文件夹存放文章；根目录提供 **`blog-index.json` 总配置文件**，统一维护列表索引，无需每次遍历全部文件夹
- 核心功能：
  - 新建/编辑文章
  - 文章列表（从 `blog-index.json` 读取，分页、筛选、搜索）
  - 手动保存（草稿 / 发布）→ 写入 **`saved/`**（已保存区）
  - 自动草稿（定时）→ 写入 **`temp_drafts/`**（临时草稿区）
  - 图片上传（按文章标题文件夹内的 `assets/` 存储）
  - 删除文章（二次确认）

## 1.2 技术栈
- 前端：Vue3 + md-editor-v3 + axios
- 后端：Node.js + Express
- 文件处理：fs/promises、path、multer

# 二、功能详细需求
## 2.1 博客列表页面
### 页面功能
- 展示 **已保存区** 与 **临时草稿区** 中的全部文章（合并列表）
- 临时草稿在列表中标注「临时」标签（仅落在 `temp_drafts/`、尚未手动保存到 `saved/` 的条目）
- 操作：编辑、删除
- 顶部固定「新建文章」按钮
- 无数据显示空状态

### 展示字段
- 文章标题（点击跳转编辑）
- 来源：已保存 / 临时草稿
- 状态：草稿 / 已发布（仅 `saved/` 内可有「已发布」；`temp_drafts/` 内均为草稿）
- 创建时间：YYYY-MM-DD HH:mm
- 更新时间：最后保存时间
- 操作栏：编辑、删除

### 列表数据来源
- **优先读取**根目录 `blog-index.json` 中的 `articles` 列表（见 3.6），在内存中完成分页、筛选、搜索
- 索引缺失或损坏时，服务端自动全量扫描 `saved/`、`temp_drafts/` 重建索引后再返回列表

### 列表能力
- 分页：默认 10 条/页
- 筛选：全部 / 草稿 / 已发布 / 仅临时草稿
- 搜索：按标题模糊匹配（对索引字段 `title` 匹配）
- 删除：二次确认弹窗，防止误删

### 路由
`/blog/list`

## 2.2 博客新增 / 编辑页面（核心）
### 页面结构
1. **顶部操作栏**
   - 返回列表
   - 页面标识：新建 / 编辑
   - 保存草稿（手动）→ 落盘到 `saved/`
   - 发布文章 → 落盘到 `saved/`
2. **文章信息区**
   - 标题输入框（必填，最大 60 字）；**文件夹名与标题一致**（经服务端安全处理，见 3.5）
3. **编辑器区（md-editor-v3）**
   - 双栏模式（编辑 + 预览）
   - 代码高亮、全屏、字数统计
   - 图片上传
4. **底部提示**
   - 自动保存状态提示（写入 `temp_drafts/`）

### 路由
- 新增：`/blog/create`
- 编辑：`/blog/edit/:id`（`id` 为 `meta.json` 内稳定标识，不随文件夹改名而变）

## 2.3 文章保存功能
### 两种业务状态（写在 `meta.json`）
| 操作 | 目标总文件夹 | `status` | 说明 |
|---|---|---|---|
| 自动保存（定时） | `temp_drafts/` | `draft` | 临时草稿，防丢稿 |
| 手动「保存草稿」 | `saved/` | `draft` | 正式保存的草稿 |
| 手动「发布文章」 | `saved/` | `published` | 已发布 |

### 单篇文章目录结构（两个总文件夹内结构相同）
```
{文章标题}/                 # 文件夹名 = 标题经 3.5 处理后的结果
├─ meta.json               # 含 id、title、status、folderName、createTime、updateTime
├─ content.md              # Markdown 原文
└─ assets/                 # 图片
```

### 保存规则
- **标题必填**才能落盘；自动保存时若标题为空，文件夹名使用 **`未命名`**，并在 `meta.json.title` 中同步，用户补全标题后下次保存**重命名文件夹**
- 内容可为空
- **手动保存 / 发布**：
  - 在 `saved/{文章标题}/` 创建或更新文件
  - 若 `temp_drafts/` 中存在同一 `id` 的临时目录，**删除该临时目录**（避免重复）
- **修改标题**：若标题变化导致文件夹名变化，服务端**重命名**对应 `{文章标题}/` 目录，并更新 `meta.json` 中的 `title`、`folderName`，**同步更新** `blog-index.json` 中该条记录的 `title`、`folderName`、`path`
- 每次保存 / 删除 / 发布 / 临时目录清理成功后，**同步更新** `blog-index.json`
- 保存成功提示：保存成功；发布成功

### 提交字段
```json
{
  "id": "文章ID（新建为空）",
  "title": "文章标题",
  "content": "Markdown 原文",
  "status": "draft | published",
  "saveMode": "auto | manual",
  "createTime": "时间戳",
  "updateTime": "时间戳"
}
```
- `saveMode: auto` → 只写 `temp_drafts/`
- `saveMode: manual` → 只写 `saved/`（`status` 由按钮决定 draft / published）

---

## 2.4 草稿自动保存（防丢核心）
### 功能说明
- 编辑过程中定时调用保存接口，`saveMode: auto`，写入 **`temp_drafts/{文章标题}/`**
- 刷新或重新进入编辑页：按 `id` 查找，优先读 `saved/`；若无则读 `temp_drafts/`
- 用户点击「保存草稿」或「发布」后，内容进入 `saved/`，并清理同 `id` 的临时目录

### 规则
- 自动保存频率：3 秒（内容有变化才请求）
- 新建文章：首次自动保存生成 `id`，在 `temp_drafts/未命名/`（或已有标题则用标题文件夹）创建目录
- 自动保存失败：红色提示，编辑区内容保留在内存中

### 状态提示
- 正在自动保存到临时草稿…
- 已自动保存到临时草稿（时间）
- 自动保存失败

---

## 2.5 图片上传功能
- 编辑器内选择图片一键上传
- 图片存入**当前文章所在标题文件夹**下的 `assets/`（可能在 `saved/` 或 `temp_drafts/`）
- 请求需带 `articleId`，服务端按 `id` 定位目录（先查 `saved/`，再查 `temp_drafts/`）
- 返回可直接访问的 URL
- 限制：png / jpg / jpeg / webp，单张 ≤ 5MB

# 三、Node.js 服务端 & 本地存储规则
## 3.1 本地目录结构
```
blog_storage/                    # 存储根目录（可配置 blogRootDir）
├─ blog-index.json               # 总配置文件：文章列表索引（列表页主要数据源）
├─ saved/                        # 总文件夹一：已保存（手动草稿 + 已发布）
│  └─ {文章标题}/
│     ├─ meta.json
│     ├─ content.md
│     └─ assets/
└─ temp_drafts/                  # 总文件夹二：临时草稿（仅自动保存）
   └─ {文章标题}/
      ├─ meta.json
      ├─ content.md
      └─ assets/
```

## 3.2 服务端运行配置（Node 项目内）
与应用代码同目录，**不放在** `blog_storage` 内，仅配置路径与行为：

```javascript
// config.js（项目根目录，示例）
module.exports = {
  blogRootDir: './blog_storage',   // 文章存储根路径
  savedDirName: 'saved',
  tempDraftDirName: 'temp_drafts',
  indexFileName: 'blog-index.json', // 总配置文件名，位于 blogRootDir 下
  autoCreateDir: true               // 启动时自动创建根目录、两个总文件夹及空索引
}
```

## 3.3 文章 ID
- 使用 UUID 或时间戳随机字符串，写入 `meta.json.id`
- **文件夹名用标题，不用 ID**；列表、编辑、删除、上传等接口仍以 `id` 定位文章

## 3.4 服务端核心逻辑
- **保存文章**：根据 `saveMode` 写入对应总文件夹下的 `{文章标题}/`；手动保存后删除同 `id` 的临时目录 → **更新 `blog-index.json`**
- **获取列表**：读取 `blog-index.json` → 筛选 / 搜索 / 分页后返回；索引不可用时执行**重建索引**（见 3.6）再返回
- **获取详情**：优先根据 `blog-index.json` 中该 `id` 的 `path` 定位目录，再读 `content.md`；索引无记录时回退为按 `id` 扫描两个总文件夹
- **删除文章**：删除标题文件夹 → 从 `blog-index.json` 移除对应条目
- **图片上传**：根据索引中的 `path` 或按 `id` 定位 → 写入 `assets/`
- **标题改名**：重命名文件夹并更新文章 `meta.json` 与索引中该条的 `title`、`folderName`、`path`、`updateTime`

## 3.6 总配置文件 `blog-index.json`
位于 `blogRootDir` 根目录，与 `saved/`、`temp_drafts/` 同级，**集中存放列表及定位信息**，便于接口快速读取，也可供本机其他工具直接打开查看。

### 文件结构示例
```json
{
  "version": 1,
  "updatedAt": 1716969600000,
  "articles": [
    {
      "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
      "title": "我的第一篇文章",
      "folderName": "我的第一篇文章",
      "source": "saved",
      "status": "published",
      "path": "saved/我的第一篇文章",
      "createTime": 1716900000000,
      "updateTime": 1716969600000
    },
    {
      "id": "temp-uuid-xxxx",
      "title": "未命名",
      "folderName": "未命名",
      "source": "temp",
      "status": "draft",
      "path": "temp_drafts/未命名",
      "createTime": 1716960000000,
      "updateTime": 1716965000000
    }
  ]
}
```

### 字段说明
| 字段 | 说明 |
|---|---|
| `version` | 索引结构版本，便于后续升级 |
| `updatedAt` | 索引最后更新时间戳 |
| `articles` | 文章索引数组 |
| `articles[].id` | 文章唯一 ID，与单篇 `meta.json.id` 一致 |
| `articles[].title` | 展示标题 |
| `articles[].folderName` | 实际文件夹名（含重名后缀） |
| `articles[].source` | `saved`（已保存区）或 `temp`（临时草稿区） |
| `articles[].status` | `draft` / `published` |
| `articles[].path` | 相对 `blogRootDir` 的文章目录路径，如 `saved/标题` |
| `articles[].createTime` | 创建时间 |
| `articles[].updateTime` | 最后更新时间 |

### 维护规则
- **写入时机**：每次自动保存、手动保存、发布、删除、标题重命名、临时目录清理后，立即更新索引（先写临时文件再原子替换，避免写一半损坏）
- **同一 `id` 仅一条**：从 `temp` 迁入 `saved` 时，更新原条目（改 `source`、`path`、`status` 等），不保留重复 `id`
- **手动保存清理临时稿**：删除 `temp_drafts` 下目录后，索引中删除 `source: temp` 的同 `id` 记录（或合并为 `saved` 一条）
- **启动检查**：服务启动时若不存在 `blog-index.json`，创建空索引 `{ "version": 1, "updatedAt": ..., "articles": [] }`
- **重建索引**：遍历 `saved/*`、`temp_drafts/*` 下各目录的 `meta.json`，全量生成 `articles` 并写回；用于索引丢失、被手动改坏、与磁盘不一致时修复

### 读取方式
- **Web 列表接口**：读索引 → 内存分页 / 筛选 / 搜索
- **本机直接使用**：用编辑器或脚本打开 `blog_storage/blog-index.json` 即可查看全部文章摘要，无需进入各标题子文件夹

## 3.5 文件夹名（= 文章标题）处理规则
- 文件夹名与文章标题一一对应，保存前经服务端统一处理：
  - 去掉首尾空格
  - 将非法文件名字符 `\ / : * ? " < > |` 替换为 `_`（或删除）
  - 长度上限建议 80 字符，超出截断
- **重名**：同一总文件夹（`saved` 或 `temp_drafts`）内若已存在同名文件夹且 `id` 不同，在文件夹名后追加 `_2`、`_3`…；`meta.json.folderName` 记录实际目录名，`meta.json.title` 保持用户原始标题
- 路径安全：只允许访问 `blogRootDir/saved/`、`blogRootDir/temp_drafts/` 下路径，禁止 `..` 穿越

# 五、完整接口定义（Node.js 本地文件版）
| 接口 | 方法 | 参数 | 功能 |
|---|---|---|---|
| /api/blog/list | GET | page,size,status,keyword,source | 从 `blog-index.json` 读取并分页；`source`：`saved` / `temp` / 全部 |
| /api/blog/index | GET | — | 返回完整 `blog-index.json` 内容（供调试或外部工具） |
| /api/blog/rebuild-index | POST | — | 扫描磁盘重建 `blog-index.json` 并返回 |
| /api/blog/detail/:id | GET | id | 按 id 获取详情（自动查 saved 与 temp_drafts） |
| /api/blog/save | POST | id,title,content,status,saveMode | 保存；`saveMode=auto`→temp_drafts，`manual`→saved |
| /api/blog/delete/:id | DELETE | id | 删除对应标题文件夹 |
| /api/blog/upload/img | POST | file,articleId | 按 id 上传到对应 assets |

# 六、非功能需求
- 性能：列表读索引，避免每次全目录遍历；自动保存不卡顿；页面加载 < 1.5s
- 可用性：刷新后按 id 从本地目录恢复；手动保存后临时目录自动清理
- 兼容性：Chrome / Edge / Firefox；移动端适配
- 安全：图片类型与大小限制；路径校验；禁止目录穿越
- Node 版本：≥ 14.x
