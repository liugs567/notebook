# 批量导入 / 拖拽上传需求文档

## 文档说明
本文档描述在现有 **liu-blog**（Vue3 + md-editor-v3 + Node 本地文件存储）基础上，新增的**列表页 Markdown 导入**能力，用于将本地已有的 Markdown 文章快速迁入系统。

与主需求文档 `doc/需求.md` 的关系：
- **仅文章列表页**（`/blog/list`）提供导入区域；编辑页、新建页**不提供**批量导入入口
- 每个导入包在 **`saved/{文章标题}/`** 下新建一篇文章
- 正文存 `content.md`，元信息 `meta.json`，图片进 `assets/`
- 导入成功后更新 `blog-index.json`，列表自动刷新

---

# 一、背景与目标

## 1.1 使用场景

导入分为两大类，**不可混用**：

| 类型 | 说明 |
|------|------|
| **上传 Markdown** | 选择一个或多个 `.md` 文件，每个文件新建一篇文章（纯文本，不处理图片） |
| **上传文件夹** | 选择一个本地文件夹，再选子模式（见 1.2） |

## 1.2 文件夹子模式

| 模式 | 说明 | 结果 |
|------|------|------|
| **文章包** | 文件夹内 **1 个** `.md` + 若干图片（任意子目录） | 新建 **1 篇**文章，图片统一进 `assets/`，正文引用按**文件名**匹配替换 |
| **多篇 Markdown** | 文件夹内 **多个** `.md` 文件 | 每个 `.md` 各新建 **1 篇**文章，按文件名匹配复制引用到的图片 |

## 1.3 功能目标
- 列表页提供导入区域，先选导入类型（Markdown / 文件夹），文件夹再选子模式
- 导入过程有进度与结果反馈（成功 / 跳过 / 失败原因）
- 与编辑页内「单张图片上传」（`POST /api/blog/upload/img`）并存，互不替代

## 1.4 非目标（本期不做）
- 编辑页 / 新建页不提供拖拽导入
- 不支持 Markdown 与文件夹混合一次导入
- 不支持 zip 压缩包解压导入
- 不支持 Word / PDF 等非 Markdown 格式
- 不静默覆盖 `saved/` 下已有同名文章（自动追加 `_2`、`_3` 后缀）

# 二、入口与界面（仅列表页）

## 2.1 页面与布局
- **路由**：`/blog/list`（`BlogList.vue`）
- 在列表表格上方增加**常驻**导入区域
- 顶部切换：**上传 Markdown** | **上传文件夹**
- 选「上传文件夹」时显示子选项：**文章包（1 个 md + 图片）** | **多篇 Markdown**

## 2.2 区域结构示意
```
┌─ 文章列表页 ─────────────────────────────────────┐
│  [标题]                          [新建文章]       │
│  [搜索] [状态筛选] [刷新]                         │
├─ 导入 Markdown ──────────────────────────────────┤
│  ( ) 上传 Markdown  ( ) 上传文件夹                │
│      └ 文章包（1 md + 图片） / 多篇 Markdown     │  ← 仅文件夹时显示
│  ┌──────────────────────────────────────────┐  │
│  │  拖拽区 / 选择按钮                         │  │
│  └──────────────────────────────────────────┘  │
│  [待导入队列]  [开始导入]                         │
├─ 文章表格 … ─────────────────────────────────────┤
└──────────────────────────────────────────────────┘
```

## 2.3 交互要求
- **上传 Markdown**：可多选 `.md`，可多次追加，最多 20 个
- **上传文件夹**：每次仅 **1 个**文件夹；重新选择会替换队列
- 切换导入类型或文件夹子模式时，**清空**当前队列
- 用户点击「开始导入」后才提交（避免误触）
- 导入中禁用操作区，显示进度
- 完成后 `message` 汇总，**自动刷新列表**
- 结果弹窗展示每条详情（成功可跳转编辑页）

# 三、导入规则

## 3.1 上传 Markdown
- 扩展名：`.md`（大小写不敏感）
- 每个文件 → `saved/{文章标题}/` 新建一篇
- **标题**：优先取文件内第一个 `# 标题`；否则取文件名（去 `.md`）
- **图片**：不处理；正文内图片路径保持原样

## 3.2 文件夹 · 文章包模式
- 必须包含 **恰好 1 个** `.md`；多个 md 时报错，提示改用「多篇 Markdown」
- 递归收集文件夹内所有图片（`png/jpg/jpeg/webp/gif`）
- 图片复制到该文章 `assets/`，重名追加 `_2`、`_3`
- **路径替换（简化）**：正文中的相对路径图片，只要**文件名**与包内某张图片相同，即替换为 `./assets/{文件名}`

示例：

| 正文引用 | 包内文件 | 替换为 |
|---------|---------|--------|
| `assets\cmd.png` | 任意目录下的 `cmd.png` | `./assets/cmd.png` |
| `./images/a.png` | 任意目录下的 `a.png` | `./assets/a.png` |

- 外链、绝对路径不改动
- 找不到同名图片时**静默保留**原路径，不弹警告

## 3.3 文件夹 · 多篇 Markdown 模式
- 文件夹内每个 `.md`（含子目录）各新建一篇文章
- 每篇文章仅复制其正文**引用到的**图片（按文件名匹配）到各自 `assets/`
- 路径替换规则同 3.2

## 3.4 落盘与冲突
| 项 | 规则 |
|----|------|
| 写入位置 | `saved/{文章标题}/` |
| 初始状态 | `status: draft`，`source: saved` |
| 同名文章 | 自动追加 `_2`、`_3` 后缀创建（与手动保存一致） |
| 单 md 大小 | ≤ 2MB |
| 单图大小 | ≤ 5MB（超出则跳过该图） |
| 单次 md 数 | ≤ 20 |
| 单次总大小 | ≤ 50MB |

# 四、接口设计

## 4.1 批量导入
```
POST /api/blog/import
Content-Type: multipart/form-data
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| files | File[] | 是 | 文件列表；文件夹用 `webkitRelativePath` |
| mode | string | 是 | `md` / `folder-package` / `folder-multi-md` |
| status | string | 否 | 默认 `draft` |

**响应示例：**
```json
{
  "code": 0,
  "data": {
    "results": [
      {
        "name": "day01.md",
        "type": "file",
        "status": "success",
        "articleId": "uuid-1",
        "title": "day01",
        "path": "saved/day01"
      }
    ],
    "successCount": 1,
    "skippedCount": 0,
    "failedCount": 0
  },
  "message": "导入完成：成功 1，跳过 0，失败 0"
}
```

# 五、验收标准

- [ ] 仅列表页可见导入区
- [ ] 「上传 Markdown」：多 md 各建一篇，不处理图片
- [ ] 「文章包」：1 md + 图片文件夹，图片进 `assets/`，`assets\xxx.png` 类引用可正常预览
- [ ] 「多篇 Markdown」：文件夹内每个 md 各建一篇
- [ ] 不可 md 与文件夹混合导入
- [ ] 同名文章自动加后缀创建，列表自动刷新

# 六、典型示例

### 示例 A：Typora 文章包
```
day01/
├─ day01.md          ← 引用 assets\cmd.png
└─ assets/
   └─ cmd.png
```
→ 选「上传文件夹 · 文章包」→ 新建 1 篇，图片可用

### 示例 B：多个 md
```
notes/
├─ a.md
├─ b.md
└─ c.md
```
→ 选「上传文件夹 · 多篇 Markdown」→ 新建 3 篇

### 示例 C：纯 md
```
a.md、b.md
```
→ 选「上传 Markdown」→ 新建 2 篇
